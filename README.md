# auth_service
***
RBAC + ABAC auth service pattern with other api's integration

***

## Назначение
Сервис авторизации. Изначально планировался как уникальный шаблон 
для однотипных сервисов авторизации.

***

## Архитектура сервиса:

- api - ручки, схемы к ним
- db - настройки сессии бд и утилиты
- docker - dockerfile, скрипт для запуска тестов
- lib - сторонние сервисы, логгирование 
- migrations - миграции
- service - основная логика по ручкам
- tests - тесты по функционалу

***

### Фичи

- Есть возможность выдать как общие права пользователю по роли, так и точечно
- Пользователи могут входить в группы(организация -> департамент -> пользователь)
- Реализован полноценный класс-логгер
- Есть реализация механики взаимодействия с s3-proxy сервисом
- Есть полностью кастомный обработчик ошибок


## Запуск проекта

1. Склонировать репозиторий: 
``` git clone git@github.com:st3ir/auth_service.git```
2. Зайти в корень:
``` cd app ```
3. Запустить docker-compose:
``` docker compose -f docker-compose.yml -p auth-service up -d ```
4. Перейти в swagger http://localhost:8100/api/docs#/
